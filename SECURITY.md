## Відомі питання безпеки

- Бот — персональний; жодної автентифікації між користувачами не передбачено. Кожен Telegram-акаунт бачить тільки власні дані.
- `BOT_TOKEN` — критичний секрет. Зберігайте тільки у `.env`, ніколи не комітьте у репозиторій.

## Рекомендації з безпечного розгортання

1. Запускайте бота від непривілейованого користувача (`non-root` у Docker).
2. Файл `data/expenses.db` зберігайте на томі з обмеженим доступом.
3. Не виставляйте бота за reverse-proxy без необхідності (polling-режим не потребує відкритого порту).
4. Регулярно оновлюйте залежності: `pip list --outdated`.
5. Перевіряйте `.gitignore` перед кожним `git add`, щоб випадково не закомітити `.env` або `*.db`.
